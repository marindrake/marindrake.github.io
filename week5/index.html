    Ch10: Testing and Debugging.

    1. Bugs are unavoidable in code, and it’s best to find them early rather than later.
    2. JavaScript can be put into strict mode using the string "use strict" . This can be used in a whole file or just a single function.
    3. Linting tools can be used to ensure your code follows good practice and conventions.
    4. Feature detection can check whether a method is supported before calling it, helping to avoid an exception being thrown.
    5. The console and browser’s built-in debugging tool can be used to interactively find and fix bugs in code.
    6. Exceptions can be thrown using the throw statement.
    7. An error object is created when an exception occurs.
    8. Any code placed inside a try block will pass any error objects to a catch block when an exception occurs. Any code inside a finally block will run if an exception does or does not occur.
    9. Test-driven development is the practice of writing tests that fail, then writing the code that passes the test, then refactoring the code every time a new feature is implemented.
    10. The Jest framework can be used to test your code.
